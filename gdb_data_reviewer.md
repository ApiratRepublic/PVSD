## จุดประสงค์
สำหรับตรวจสอบรูปแบบข้อมูลที่อยู่ในฟีเจอร์คลาสต่าง ๆ ว่าตรงตามข้อกำหนดหรือไม่
ข้อกำหนด อ่านได้ที่ GDB Data Standard.md

## สิ่งที่ต้องมี
ไพธอนสคริปต์นี้ ผู้ใช้ต้องติดตั้ง arcpy และ pandas ก่อนใช้งานสคริปต์นี้
สามารถรันได้ใน Python environment ที่มี arcpy (เช่น ArcGIS Pro) 

## การตั้งค่า
-	gdb ทั้งหมด เก็บไว้ใน root directory เดียวกัน
-	ค่าที่ต้องกำหนดในตอนต้นของไฟล์
	-	ROOT_DIR = ที่รวมไฟล์ GDB
	-	REPORT_ROOT = ที่เก็บรายงานผลเป็นเอ็กเซล (รายละเอียดสิ่งที่ไม่ตรงมาตรฐาน)
	-	OVERLAP_ROOT = ที่เก็บไฟล์ผลการตรวจสอบทับซ้อน
	-	SUMMARY_EXCEL_PATH = ใส่พาร์ธไฟล์เอ็กเซลสรุปรายงานรวม

ROOT_DIR: ที่เก็บ GDB ทั้งหมด ภายในควรกำหนดโครงสร้างโฟลเดอร์ให้เหมือนกัน 
เช่น 49-มุกดาหาร\GDB_49_2\data.gdb
เช่น 10_1-กรุงเทพมหานคร1\GDB_10_1\data.gdb
ใช้ขีดกลาง "-" หรือ ขีดล่าง "_" คั่นระหว่างรหัสจังหวัดกับชื่อจังหวัด (เพราะง่ายต่อการแยกชื่อจังหวัดตอนทำรายงาน)
